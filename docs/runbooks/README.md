# Runbooks

One per service. Each runbook covers:

- **Purpose** — what the service does, in one paragraph.
- **SLOs** — latency, availability, throughput.
- **Dashboards & alerts** — links to Grafana, paging policy.
- **Runbook entries** — one per known alert type, with diagnosis and remediation steps.
- **Deploy** — how to ship, how to roll back.
- **Contacts** — service owner, on-call, escalation.
